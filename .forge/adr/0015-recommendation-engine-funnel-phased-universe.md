# 추천 엔진 — 2단 깔때기·점진 유니버스·정량 플래그 근거(LLM 0)

## Context

종목 [[추천]] 신규 기능을 만든다. 발굴 범위는 **전체 시장**(~2700 KR + 503 US)이 목표지만, 추천 점수의 근거가 되는 *리치 신호*(컨센서스/목표가·KR 수급·지분공시)는 모두 **종목별 외부 호출**이다 — 키움 throttle 0.25s/콜, DART corp_code별, Naver/FnGuide per-stock. 전 종목에 매일 계산하면 일배치가 수십 분~시간이 된다. 또 "왜 추천"을 AI 내러티브로 쓰면 분석은 외부 Claude Cowork(LLM)가 써야 하고, 후보 수에 비례하는 토큰·실행 비용이 든다(백엔드엔 LLM이 없다). 즉 *발굴 범위(넓게)·근거 풍부함·배치 비용*이 삼각 트레이드오프다.

## Decision

1. **2단 깔때기.** 싼 *시장 전체* 신호(당일 가격·거래대금·거래량·시총 + US 구루 멤버십)로 [[발굴 유니버스]] 후보를 좁힌 뒤, **상위 K개 후보에만** 비싼 리치 신호(OHLC 모멘텀·컨센서스·KR 수급/지분공시)를 enrich한다.
2. **점진 유니버스.** v1 = KR 시총 상위 ~200(Naver 스냅샷 재사용) + US S&P500(503) + 전 유저 추적종목(+US 구루 보유). 시총 종목수가 성장 다이얼 — 전체 시장으로 단계 확장.
3. **점수 = 멀티팩터 합성 0~100 + 정량 플래그.** 밸류(목표가 상승여력)·모멘텀·스마트머니를 투명 가중으로 합성해 정렬 가능한 점수를 내고, "왜 추천"은 기여 큰 정량 플래그로 보인다. **백엔드 LLM 호출 0.**
4. **AI 근거는 [[딥다이브]]로 on-demand·비동기.** 딥다이브 = 관심종목 추가 → 기존 "추적 시 Cowork 분석" 흐름이 다음 run에 채운다. 추천 기능은 Cowork 큐를 신설하지 않는다.
5. **batch precompute → 저장 → 요청은 read.** `recommendation_kr`(20:30 KST)·`recommendation_us`(07:00 KST) 시장 분리 배치가 점수를 사전계산해 `stock_recommendations`에 저장하고, `GET /api/recommendations`는 저장값만 읽는다(요청경로 외부 호출 0). 시장 분리는 ADR-0012/0013 연장.
6. **세 섹션은 글로벌 점수의 per-user partition.** 발굴 = 점수 유니버스 − 호출자 추적종목, 관심 재정렬 = ∩ 호출자 watchlist, 보유 액션 = ∩ 호출자 holdings(+ 포지션 맥락 액션). 점수는 종목 공유, 섹션은 요청 시 분기.

## Why / Trade-off

- **전 종목 리치 계산(대안 C) 거부**: 키움 throttle·DART·yfinance 쿼터로 일배치가 수십 분~시간. 깔때기는 발굴 범위(전체)와 근거(리치)를 *비용 한도 안에서* 양립시킨다. "요청·기동 경로 라이브 fetch 금지 / 배치 precompute" 패턴(task#48·50)의 연장.
- **AI 근거 사전생성(대안 C') 거부**: Cowork 토큰/실행 비용이 매일 후보 수에 비례한다. 정량 플래그는 비용 0·유니버스 확장 무제한·추적 가능('wrong < missing'의 투명성 — ADR-0014 플래그→밴드 원칙의 연장). 풍부한 AI 분석은 사용자가 *원하는* 종목에만(딥다이브) 든다.
- **점진 유니버스**: 작은 유니버스엔 리치 신호를 거의 다 깔 수 있어 콜드스타트가 쉽고, 커질수록 깔때기가 Stage-2를 제한한다. 소형주 데이터 품질·대량 다운로드 쿼터 리스크를 단계적으로 흡수.
- **Cowork = 관심추가 위임**: 분석 큐 서브시스템 신설을 회피하고 기존 흐름을 재사용. "오래된 리포트 순 자동 갱신 큐"는 *모든* 리포트에 걸리는 별도 니즈라 본 기능 밖(별도 task 후보).

## Consequences

- **미추적 종목은 리치 신호가 결측**일 수 있다(KR 수급/지분공시·컨센서스는 추적종목 위주). 가용 팩터만으로 점수(graceful degrade)하고 결측을 표시한다. 추적(관심추가)되면 다음 배치에 풀 점수로 승격. **(갱신: 이 graceful-degrade 합성=결측군 재정규화는 ADR-0016이 중립 채움(denom 항상 1.0, 단일 축 만점 불가)으로 대체. 결측 표시는 유지.)**
- **구루 팩터는 US 전용**(dataroma 13F) → KR 발굴은 가격·거래·모멘텀·(추적 시)수급/지분공시로만 점수.
- **유니버스 성장 = 시총수 상향**이고 배치 비용이 선형 증가한다. yfinance 대량 다운로드 쿼터와 소형주 데이터 품질이 확장 상한.
- `stock_recommendations`는 종목 공유(per-ticker) 테이블 — 발굴은 추적 제외 필터, 관심/보유는 추적 교집합으로 같은 저장값을 재사용.
- 추천 플래그/액션 배지 색은 [[수급 스코어]]·[[지분공시 신호]]와 동일하게 **KR 가격 토큰(`success`/`danger`) 금지·전용색 명시**(가격색 반전 트랩, b288f494).
