# ADR-0025 — 디자인 아이덴티티: 프로 금융 터미널(다크 기본) + 좌측 사이드바 IA

- 상태: 채택
- 날짜: 2026-07-11
- 관련: task#171~175(디자인 전면 리뉴얼 5파트), ADR-0011(플랫폼)·기존 KR 색 관례 가토

## 맥락

PortfoliOn의 화면은 기능 단위로 증축돼 아이덴티티가 흐릿하다(범용 라이트 카드 UI + 상단 nav + 리서치 허브에 7하위탭 과밀). 사이트 주제(개인 투자 리서치·지표 터미널)에 맞는 전면 리뉴얼을 그릴링한 결과, 색·타이포·밀도·레이아웃 골격을 지배하는 최상위 결정 세 가지가 나왔다.

## 결정

1. **디자인 아이덴티티 = 프로 금융 터미널**: 다크 기본·데이터 밀도 높음·모노스페이스 숫자·차분한 액센트. 라이트 테마는 토글로 유지(기본 테마만 다크로 전환, 기존 사용자 설정 존중).
2. **PC 골격 = 좌측 사이드바**(아이콘+라벨, 5섹션: 리서치/포트폴리오/시장/일정·인컴/구루 + 하단 설정·admin). 상단 가로 nav 폐지. 모바일은 하단 MobileNav 유지.
3. **사이드바 항목 = 개별 URL 라우트**(딥링크·새로고침 복원). 허브 내부탭 상태는 라우트로 이관, 구 URL은 리다이렉트.

경계(변경하지 않는 것): **KR 가격색 관례 유지**(`--up`=빨강 상승/`--down`=파랑 하락 — 채도만 다크 대비로 재조율), **메뉴 권한은 기존 허브 단위 유지**(`ALL_MENUS` 5키 불변 — 사이드바 섹션이 기존 키에 매핑, 일정·인컴 섹션=research 권한), **plain CSS 유지**(Tailwind 등 프레임워크 도입 없음).

## 근거

- 터미널 방향은 사이트 주제(지표·리포트·수급 데이터 밀도)와 현행 화면 밀도에 부합 — 클린 핀테크(라이트·여백 넓음)는 고밀도 화면과 긴장.
- 사이드바는 세로 공간을 데이터에 양보하면서 리서치 7하위탭 과밀을 섹션 승격으로 해소하는 유일한 골격 — 상단 nav 유지+재그룹핑으로는 과밀이 남는다.
- 개별 라우트는 사이드바 문법의 자연 귀결(항목=목적지). 내부탭 상태 유지는 딥링크 불가를 영속화.

## 결과

- 전 화면 CSS·레이아웃이 이 결정 위에 재구축된다(5파트) — 이후 방향 변경은 전면 재작업이라 비싸다(이 ADR을 남기는 이유).
- 권한·백엔드 스키마는 무변(허브 단위 유지)이라 서버 영향 0.
- 기존 사용자: 기본 테마가 다크로 바뀌고(라이트 토글 가능) 상단 nav가 사이드바로 바뀌는 동선 변경을 감수.

## 검토한 대안

- **클린 핀테크(라이트 기본)**: 기각 — 데이터 밀도 높은 화면과 긴장, 주제 부합도 낮음.
- **다크 온리**: 기각 — 기존 라이트 사용자 경험 강제 변경, 유지비 절감 대비 손실 큼.
- **상단 nav 유지 + 재그룹핑**: 기각 — 리서치 과밀 미해소, 세로 공간 손해.
- **하이브리드(글로벌바+사이드바)**: 기각 — 작업량 최대 대비 증분 가치 낮음(YAGNI).
