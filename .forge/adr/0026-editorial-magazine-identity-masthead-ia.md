# ADR-0026 — 디자인 아이덴티티 교체: 에디토리얼 매거진(라이트 기본) + 마스트헤드 IA

- 상태: 채택 (ADR-0025의 아이덴티티·골격 결정을 대체)
- 날짜: 2026-07-17
- 관련: ADR-0025(대체 대상), task#190~194(에디토리얼 전면개편 5파트)

## 맥락

ADR-0025(2026-07-11)로 "프로 금융 터미널(다크 기본) + 좌측 사이드바"가 채택돼 전 화면이 재구축됐으나, 6일 사용 후 사용자가 네 가지 이유로 교체를 결정했다:

1. **몰개성·차별화 부족** — 다크 터미널은 투자 앱의 흔한 문법이라 기성품 느낌.
2. **다크·고밀도가 부담** — 장시간 읽기에 다크 고밀도 화면이 피로.
3. **'리서치' 주제 부합** — 이 앱의 본질은 실시간 트레이딩이 아니라 읽고 생각하는 리서치 — 터미널보다 매거진이 정체성에 맞음.
4. **시각적 재미·감성 부족** — 애니메이션·일러스트로 생동감을 원함.

## 결정

1. **디자인 아이덴티티 = 에디토리얼/매거진**: 종이질감 **라이트 기본**(다크는 '야간 인쇄본' 토글로 유지, 기존 사용자 명시 설정 존중), 제목만 세리프(Noto Serif KR 서브셋 — 본문·데이터는 산세리프 + tabular 숫자 유지), 손그림 잉크 선화 SVG 일러스트(빈상태·에러, 카테고리 아이콘, 로그인 히어로, 절제된 차트 장식 — 직접 저작·번들 포함).
2. **PC 골격 = 매거진 마스트헤드**(상단 제호 + 가로 카테고리 nav 5섹션). 좌측 사이드바 폐지. **URL 라우트는 전부 유지**(딥링크·리다이렉트 무변 — 골격 스킨만 교체). 모바일은 하단 MobileNav 유지, 스킨만 교체.
3. **모션 = CSS 우선 + 최소 JS**(라이브러리 무추가): 페이지 진입/스크롤 reveal, 스케치 stroke-dashoffset 드로잉, 마이크로 인터랙션, 숫자 카운트업(첫 진입 1회). `prefers-reduced-motion` 존중.

경계(ADR-0025에서 계승, 불변): **KR 가격색 관례 유지**(`--up`=빨강/`--down`=파랑 — 채도만 종이질감에 맞게 잉크톤 재조율), **메뉴 권한 `ALL_MENUS` 5키 불변**(마스트헤드 카테고리가 기존 키에 매핑), **plain CSS 유지**(Tailwind·모션 라이브러리 도입 없음).

## 근거

- 에디토리얼은 "읽는 리서치 앱"이라는 주제와 정합하고, 손그림 스케치는 흉내 내기 어려운 차별화 자산.
- 라이트 기본은 장시간 읽기 피로를 줄이고 종이 메타포와 일치. 다크 토글 유지로 기존 다크 사용자 손실 없음.
- 라우트·권한·백엔드 무변이라 교체 비용은 표시계층에 국한 — 사이드바→마스트헤드는 nav 스킨 교체이지 IA 재설계가 아님.

## 결과

- 전 화면 CSS·레이아웃이 다시 재구축된다(5파트, task#190~194). 이후 방향 변경은 또 전면 재작업 — 이 ADR의 교체 사유 4건이 다음 방향 논의의 출발점이다.
- 6일 만의 방향 전환 비용을 감수한 의식적 결정: 터미널 5파트 작업물 중 토큰 인프라·프리미티브 단일소비·라우트 승격·감사 인프라는 그대로 재사용된다(버려지는 것은 비주얼 스킨).

## 검토한 대안

- **터미널 유지 + 모션/폴리시 진화**: 기각 — 교체 사유 1·2·3(정체성 자체의 불만)을 해소 못 함.
- **다크 기본 에디토리얼**: 기각 — 종이질감·스케치 표현력이 제한되고 사유 2(다크 부담) 미해소.
- **골격(사이드바) 유지 + 비주얼만 교체**: 기각 — 사이드바는 터미널 문법이라 매거진 언어와 긴장, 사용자가 골격 교체를 명시 선택.
- **모션 라이브러리(framer-motion) 도입**: 기각 — 에디토리얼 모션은 페이드·드로잉 중심이라 CSS로 충분, 의존성·번들 증가 불필요(YAGNI).
