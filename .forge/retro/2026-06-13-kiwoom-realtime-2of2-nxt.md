# 2026-06-13 — NXT 확장시간 KR 시세 (Phase 3 part 2/2)

## Plan vs actual
- What went as planned: 3 슬라이스(S1 라이브 프로브 → S2 통합코드 → S3 폴링 윈도우) 구현. get_quote_kr/get_history_df가 통합가 반환(005930 324,500), 비-NXT KRX 폴백, 폴링 08:00~20:00. 494 테스트·빌드·라이브 e2e·가상 UAT(확장시간 폴링) PASS. main 8f3b0523 배포.
- Divergences (낮음~중간):
  - **S1 프로브가 S2를 단순화** — 계획은 "NXT 미거래 종목 멤버십 체크 후 KRX 폴백"이었으나, 프로브에서 **통합(`_AL`)이 전 종목 안전**(비-NXT는 통합이 KRX로 자동 폴백) 확인 → 멤버십 체크 불필요, `_AL` 접미사만 부착하면 끝. `_NX`(NXT전용)는 비-NXT 빈응답+차트 298봉뿐이라 미사용.
  - **통합 채택으로 KR 표시가가 NXT-inclusive로 이동**(322,500→324,500) — ADR-0010으로 승급.
  - **isKrMarketOpen 확장(08~20)이 part1 폴링도 확장** — 같은 함수라 soft 결합(의도).

## Learnings
- Do differently next time:
  - **딥리서치가 "미검증"으로 남긴 외부 API 동작은 라이브 프로브로 닫는다** — NXT 리서치는 "통합=합성 BBO, 합산 아님"까지 줬지만 "키움 REST `_AL` 응답이 실제로 뭘 주는지"는 open question으로 남겼다. 코드 전(前) **단일 슬라이스 라이브 프로브**(여러 접미사×여러 종목 1콜 비교)로 닫으니 S2 설계가 확정·단순화됐다. **리서치=방향, 프로브=확정**. 불확실 외부 API는 "프로브 슬라이스 선행"을 계획에 박는 게 정답이었다.
  - **'전 종목 안전한 단일 코드'를 프로브로 찾으면 분기 로직이 사라진다** — 비-NXT에 `_AL`이 KRX로 자동 폴백한다는 한 가지 사실이 "종목별 NXT 멤버십 테이블/체크"라는 계획상 큰 덩어리를 통째로 제거했다. 분기 설계 전에 "더 너그러운 단일 입력이 있나"를 프로브로 먼저 묻기.
  - **시간 게이팅 함수는 한 곳이라 윈도우 변경이 상위 기능에 전파됨** — isKrMarketOpen을 08~20으로 넓히니 part1 자동폴링도 확장시간을 커버. 같은 유틸을 공유하는 기능 간 결합을 인지하고 변경할 것(여기선 의도적 정렬).
  - 검증은 층별로: 통합가 반환(S2)은 라이브 e2e, 폴링 윈도우(S3)는 [[reference-frontend-uat]]의 Date고정 가상 UAT. 휴장일에도 종단 확인 가능. NXT 실가격 변동만 평일 확장세션 대기.
