# 2026-06-21 — KR 시세 소스 근본추적 + 전일종가 ±30% 방어로직 (task#94)

## Plan vs actual
- What went as planned: 슬라이스 1(소스별 raw 프로브) 그대로 — 백엔드 컨테이너(`portfolion-backend-1`)에서 005930 각 소스 원값을 떠 근본원인 특정.
- Divergences:
  - **슬라이스 2 방향 전환(사용자 지시)**: 플랜은 "조건부 소스측 정규화 수정"이었으나, 프로브가 *영속* 소스 버그 없음(70k는 일시적)을 보여, 사용자가 **"전날 종가 대비 ±30%(KR 일일 가격제한폭) 방어로직 추가"**로 전환. 소스 수정 대신 task#93 가드를 ±30%로 강화.
  - 플랜 Non-goal "발산 가드 변경(task#93 유지)"을 **사용자 명시 지시로 오버라이드** — 정당한 변경(사용자가 직접 더 강한 검증을 요구).

## Learnings
- Do differently next time:
  - **"갑자기 N배" 박제 이상값은 영속 소스 버그가 아니라 *일시적*일 수 있다 — 영속 버그를 가정해 소스부터 고치려 들지 말 것.** 005930 ~70k는 어느 소스도 *현재* 재현 안 됨 → 배치 시점 **NXT `_AL`(SOR 통합코드) 순간 이상체결**이 박제된 것. 진단 핵심은 **백엔드 컨테이너에서 KRX(`005930`) vs SOR(`005930_AL`) 원값을 직접 대조**(`docker exec -i portfolion-backend-1 python - < probe.py`) — 키움 정규화가 `_AL`을 써 NXT가를 반환함을 눈으로 확인.
  - **시세 위생 검증의 올바른 앵커 = 전일종가(prev_close/base_pric) ±30%(KR 일일 가격제한폭).** 프로브가 증명: cur_prc가 70k로 글리치돼도 base_pric(362,500)은 안정적 → 전일종가 기준 ±30%면 일시적 글리치를 정밀하게 잡는다. task#93의 "일봉 2배"는 독립 TR 교차검증으로 함께 유지(소스 prev_close까지 오염된 경우 완충) — `_price_sane`이 둘 다 통과 요구. 합법 하한가(-30%)는 false-reject 안 되게 경계 포함([0.7,1.3]).
  - **컨테이너 키 환경**: 키움/KIS는 `.env.docker`라 로컬 `.venv`엔 키 없음 → 라이브 프로브는 `portfolion-backend-1` 컨테이너에서. `docker compose -f`는 프로젝트명 불일치로 실패, 컨테이너 직접 `docker exec`가 확실.
- 후속(회고 메모만, task 미생성 — 사용자 결정): **NXT `_AL` vs KRX ~1% 괴리** — 일배치 스냅샷이 정규장 종가(354k)가 아닌 NXT 시간외가(350,500)를 박제한다. 버그 아닌 시간외 신선도 설계 부작용. 일배치(20:30, 마감 후) 스냅샷엔 KRX 정규장 종가가 더 정확할 수 있으나 시간외 신선도와 트레이드오프 → 필요해지면 fg-ask로 "일배치 KR 시세에 KRX 정규장 코드 사용" 그릴링.
