## Learnings
- Do differently next time:
  - **추출 실패 시 '안전한 기본값' 폴백이 가장 큰 오저장을 만든다.** `_table_unit`이 단위 캡션을 못 읽을 때 억원으로 폴백 → 백만원/USD천 값이 ×100~수십배 과대계상(454910 1.35조, 현대차 2024Q3 **903조**). 데이터 추출/정규화에서 "실패=기본값"은 금물, "실패=None(누락)"이 'wrong<missing'. → CLAUDE.md Gotchas 승격.
  - **한 문서에 동일 지표 표가 여러 개 공존하면 "첫 표 채택"은 비결정·위험.** 한화 문서엔 회사컬럼 합계표(연결 68조)와 단일처럼 보이는 표(44조)가 공존 → 첫-표 채택이 엉뚱한 값을 잡음. **다중엔티티는 표 단위가 아니라 문서 단위로 판정**(`_auto_backlog`가 susu 표 하나라도 다중엔티티면 문서 전체 pending).
  - **다중엔티티 신호는 `회사` 컬럼 헤더가 `종속회사` 문구보다 강건.** `종속회사` 카운트는 연결 전 분기(2024)에서 0이라 누수 → `회사` 컬럼으로 잡음.
  - **fixture 단위 테스트(9종 전부 통과)는 실데이터 케이스를 못 잡는다.** 운영 재적재가 USD천·줄바꿈 캡션·연결 전 분기를 드러냄 — 직전 retro(backlog-extract-precision)의 "배포 후 재적재 UAT" 교훈이 **재발**했고, 이번엔 계획에 S4(재적재 UAT)를 명시해 둔 덕에 실제로 잡아냄(교훈이 작동). → CLAUDE.md Gotchas 승격(재발 방지 영속화).
  - **fg-ask 단계의 실 HTML fixture 캡처가 결정적이었다.** flattened raw_text로는 colspan/rowspan·수주잔고 중간컬럼(두산에너빌)·단위 다양성(억원/백만원/USD천)을 못 봄 — 실 document.xml 표를 미리 캡처해 가정 다수를 실행 전에 교정. 단, fixture는 "선택된 표 1개"라 다중표 공존·연결전분기 같은 문서 레벨 케이스는 못 담아 UAT에서야 드러남(그래서 UAT 필수).
- 잔여/후속(저위험):
  - **현대차(005380) 11.6조 '철도' 수주잔고** — 실 공시 표 추출·검산 통과·QoQ 매끄러움(116→119→121→112→91→80조)으로 일관된 실데이터지만, 현대차의 수주잔고로 적절한 지표인지 데이터 해석은 사용자 검토 잔여(파서 오류 아님).
  - **유형2(건설 공사진행: 수주총액×잔여진행률)·유형4(한화 모회사 행 선택)** 자동화는 ADR-0003 비목표 → pending(Cowork)로 둠. 후속 작업 후보.
  - `_auto_backlog`는 다중엔티티 아닌 문서에서 여전히 "첫 susu 표" 채택 — 단일엔티티 문서에 동일지표 표가 여럿이면 잠재 모호(현재 종목군엔 무해, 검산+합계행 요구로 제한).
